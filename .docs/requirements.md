# open-dataset-cleaner 要件定義書

最終更新: 2026-06-24

## 1. 概要

Web から収集した文書（WARC / HTML / プレーンテキスト / JSONL）を入力し、
LLM の事前学習・ファインチューニング用データセットとして使える品質まで
クリーニング・スコアリング・重複除去・変換するパイプラインツール。

- 言語: Rust
- 配布形態: CLI バイナリ（`open-dataset-cleaner` / 短縮コマンド `odc`）
- 対象 OS: Windows / Linux / macOS
- 処理方式: ストリーミング（全件メモリ展開しない）+ rayon による並列処理

## 2. 目的・背景

- Web スクレイピング由来のデータは HTML ノイズ・ボイラープレート・重複・
  言語混在・SEO スパムなどを含み、そのまま学習に使うと品質を下げる。
- 既存のクリーナー（gopher rules, fastText lid 等）は個別機能の組み合わせが
  多く、スコアリングの透明性・設定の柔軟性・プラグイン拡張性に欠ける。
- 本ツールは「何が・どれだけの理由で除外/採用されたか」を統計として
  出力できることを差別化ポイントとする。

## 3. 用語

| 用語 | 意味 |
|---|---|
| レコード | 1 件の文書（URL + 本文 + メタデータ） |
| スコア | レコードに付与される 0.0〜1.0 等の品質指標値 |
| フィルタ | スコアや条件に基づきレコードを採用/除外するルール |
| チェックポイント | 中断・再開のために保存する処理進捗状態 |

## 4. 技術スタック（想定）

- 並列処理: `rayon`
- 言語検出: `lingua-rs`
- HTML 解析・本文抽出: `scraper` / `html5ever` + 独自ボイラープレート除去
  （`readability` 系アルゴリズムを参考に実装）
- WARC 読み込み: `warc` クレート相当
- Parquet 出力: `arrow` + `parquet`（または `polars`）
- 設定ファイル: TOML/YAML（`serde` + `toml` / `serde_yaml`）
- 重複除去: 完全一致は `blake3`/`xxhash`、近似重複は MinHash + LSH の自前実装
  もしくは既存クレートを評価のうえ選定
- Perplexity: `kenlm` バインディング（**`perplexity` cargo feature でオプション化**。
  デフォルト無効。有効化時のみ KenLM の動的/静的リンクが必要）
- プラグイン: `wasmtime`（WASM サンドボックス実行）
- CLI: `clap`

## 5. 入出力仕様

### 5.1 入力

- [ ] WARC ファイル（`.warc` / `.warc.gz`）
- [ ] HTML ファイル（単体 / ディレクトリ / glob）
- [x] プレーンテキスト（1ファイル1文書、または改行区切り）
- [x] JSONL（1行1レコード、フィールドはマッピング設定で指定可能）
- [x] 入力はストリーミングで逐次読み込み、全件を一度にメモリ展開しない
- [x] 複数入力ファイル/ディレクトリの一括指定（glob パターン対応）

### 5.2 出力

- [ ] Parquet 形式（スキーマ: 本文・メタデータ・各スコア列）
- [x] JSONL 形式
- [ ] 出力をシャード分割可能（最大行数/サイズで分割）
- [x] 除外されたレコードを別出力（除外理由付き）にオプションで出力できる

## 6. パイプライン概要

```
入力読込（ストリーミング）
  → HTML除去・本文抽出（ボイラープレート除去、Markdown変換オプション）
  → 言語検出・文字種解析
  → 品質スコアリング（テキスト品質・コンテンツ品質・任意でPerplexity）
  → 重複除去（完全一致 → 近似重複 MinHash/LSH）
  → フィルタリング（閾値・カスタムプラグイン適用）
  → 出力書き込み（Parquet/JSONL）+ 統計レポート更新
  → チェックポイント記録
```

## 7. 機能要件

### 7.1 テキスト抽出

- [ ] HTML タグ除去・本文抽出
- [ ] ボイラープレート除去（ナビゲーション・フッター・広告枠・サイドバー）
- [ ] Markdown 変換オプション（見出し・リスト・リンクの保持/除去を設定可能）
- [x] 抽出後テキストに残留 HTML タグ・URL が無いことを検証可能にする

### 7.2 品質スコアリング — 言語系

- [x] 言語検出（`lingua-rs`、文書単位）
- [x] 言語混合率スコア（文単位で言語判定 → 比率を算出）
- [x] 文字種比率算出（ひらがな・カタカナ・漢字・英数字・その他）

### 7.3 品質スコアリング — テキスト品質

- [x] 重複行率（文書内の行重複比率）
- [x] 記号・数字の異常比率（記号/数字が異常に多い行・文書の検出）
- [x] 平均文長・文長分散
- [ ] Perplexity スコア（KenLM 連携、`perplexity` feature 有効時のみ）
- [x] URL・HTML タグの残留チェック

### 7.4 品質スコアリング — コンテンツ品質

- [ ] 広告っぽいキーワード検出（設定可能なキーワード/正規表現リスト）
- [ ] SEO スパム検出（キーワード羅列・不自然な繰り返しパターン）
- [ ] 文章の自然さスコア（n-gram 言語モデル or 簡易統計ベース、PerplexityがOFFでも動作する軽量版を用意）

### 7.5 重複除去

- [ ] 完全一致重複除去（コンテンツハッシュベース）
- [ ] MinHash によるシグネチャ生成
- [ ] LSH による近似重複候補の高速検索
- [ ] N-gram 指紋による補助的な重複検出
- [ ] 重複除去のしきい値（類似度）を設定可能にする
- [ ] 重複除去はストリーミング処理と両立する設計にする（全件をメモリに載せない、もしくは一定バッチ/シャード単位で処理する旨を明記）

### 7.6 フィルタリング

- [x] 各スコアに対する閾値設定（採用/除外の境界値）
- [x] フィルタルールを YAML/TOML の設定ファイルで定義可能
- [ ] ルールの AND/OR 等の組み合わせ条件をサポート
- [ ] カスタムフィルタをプラグイン形式で追加可能（**WASM プラグイン**、`wasmtime` でサンドボックス実行）
  - [ ] プラグインの入出力インターフェース定義（レコードテキスト/メタデータを渡し、スコアまたは採用判定を受け取る）
  - [ ] プラグインの読み込み元をローカルファイルパスで設定ファイルから指定
  - [ ] プラグイン実行のタイムアウト/リソース制限（サンドボックスの利点を活かす）

### 7.7 運用系

- [x] 並列処理（`rayon`、スレッド数を設定/CLI引数で指定可能）
- [x] 処理統計レポート出力（入力件数・除外件数・除外理由別件数・各スコア分布など）
- [ ] 途中再開対応（チェックポイント: 処理済みファイル/オフセットを記録し再実行時にスキップ）
- [x] ログ出力（処理進捗・エラー・警告。レベル設定可能）

## 8. 設定ファイル仕様（イメージ）

```toml
[input]
format = "warc"   # warc | html | text | jsonl
paths = ["./data/**/*.warc.gz"]

[output]
format = "parquet"  # parquet | jsonl
path = "./out/dataset"
shard_max_rows = 1_000_000
write_rejected = true

[extract]
markdown = true
strip_boilerplate = true

[scoring.language]
allow = ["ja", "en"]
max_mixed_ratio = 0.2

[scoring.text_quality]
max_duplicate_line_ratio = 0.3
max_symbol_ratio = 0.15
perplexity_enabled = false   # true にすると KenLM feature が必須

[dedup]
exact = true
minhash_lsh = true
similarity_threshold = 0.85

[filters.thresholds]
min_natural_score = 0.4

[[plugins]]
path = "./plugins/ad_detector.wasm"
config = { sensitivity = "medium" }

[runtime]
threads = 0  # 0 = 論理コア数
checkpoint_dir = "./.checkpoints"
```

## 9. 非機能要件

- [ ] Windows / Linux / macOS でビルド・実行可能（Windowsで確認済み。Linux/macOSは未検証）
- [x] `perplexity` feature 無効時は KenLM への依存を一切持たない
- [ ] 大規模入力（数百GB〜TB級 WARC）でもメモリ使用量が入力サイズに対して線形に増えない
- [x] 処理は冪等（同一入力・同一設定で再実行すれば同一出力になる）
- [ ] チェックポイントからの再開時に出力の重複・欠落が発生しない

## 10. スコープ外（将来拡張候補）

- Web UI / ダッシュボード
